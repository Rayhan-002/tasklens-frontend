# Project Instructions

## Code Quality

- Write production-quality, maintainable code.
- Follow Django, Django REST Framework, Next.js, React, and TypeScript best practices.
- Keep a clear separation of responsibilities.
- Prefer reusable components and avoid duplicated code.
- Use strict TypeScript typing and avoid `any` unless absolutely necessary.
- Prefer well-maintained libraries and modern patterns.

## Workflow

- Implement only the milestone I request.
- Do not continue to the next milestone until I approve.
- When a milestone is complete:
  1. Verify the implementation works.
  2. Fix any errors found.
  3. Briefly summarize what was implemented.
  4. Wait for my approval.

## Communication

- Keep explanations concise.
- Do not explain every generated file.
- Only explain important architectural decisions.
- Focus on implementation.

## Git

- Do not initialize Git.
- Do not create commits.
- Do not push to GitHub.
- I will manage version control.

## If Unsure

- Choose the solution that best follows production software engineering practices.
- If a major architectural decision is required, stop and ask before proceeding.
